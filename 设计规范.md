## 1. 设计原则

### 1.1 视觉风格
- **主题**：红黑简约风格，高对比度
- **配色**：以红色(#e74c3c)为主色调，深色背景
- **动效**：流畅的过渡动画(300ms ease)
- **排版**：清晰的信息层级，代码字体用于辅助信息

### 1.2 交互原则
- **即时反馈**：所有可交互元素都有悬停状态
- **无障碍**：支持键盘导航，足够的对比度
- **响应式**：移动端优先，断点使用 Tailwind 默认配置

### 1.3 代码哲学
- **组件化**：单一职责，可复用
- **类型安全**：TypeScript 严格模式
- **性能优先**：静态生成，懒加载非关键资源

## 6. 组件设计

### 6.1 Svelte 5 组件规范

**Props 定义：**
```typescript
interface Props {
  variant?: 'tag' | 'category' | 'default';
  size?: 'sm' | 'md';
  label?: string;
  children?: import('svelte').Snippet;
}

let {
  variant = 'default',
  size = 'sm',
  label = '',
  children
}: Props = $props();
```

**响应式计算：**
```typescript
// 正确：直接使用表达式
const finalClass = $derived(`ui-pill ui-pill-${size}`);

// 错误：不要包裹函数
const finalClass = $derived(() => `ui-pill`); // ❌
```

**事件处理：**
```svelte
<!-- 使用 Svelte 5 的回调 props -->
<script>
  interface Props {
    onSelect?: (value: string) => void;
  }
</script>

<button onclick={() => onSelect?.(value)}>
```

### 6.2 组件分层

| 层级 | 职责 | 示例 |
|------|------|------|
| **UI** | 纯视觉，无业务逻辑 | `Button`, `Pill`, `Card` |
| **Feature** | 业务功能，可复用 | `NoteCard`, `TagCloud` |
| **Layout** | 页面结构 | `Header`, `Sidebar`, `Footer` |
| **Page** | 页面级组合 | `index.astro`, `about.astro` |

### 6.3 组件文件结构

```svelte
<!-- ComponentName.svelte -->
<script lang="ts">
  // 1. 类型导入
  import type { Note } from '@types';
  
  // 2. Props 接口
  interface Props { }
  
  // 3. Props 解构
  let { ... }: Props = $props();
  
  // 4. 派生状态
  const computed = $derived(...);
  
  // 5. 副作用
  $effect(() => { });
</script>

<!-- 模板 -->
<div class="component">
  <!-- 内容 -->
</div>

<style>
  /* 组件级样式（少用，优先用全局组件类） */
</style>
```

## 7. 类型定义

### 7.1 类型文件组织

```
src/types/
├── index.ts      # 统一导出
├── note.ts       # 笔记相关类型
└── graph.ts      # 图谱相关类型
```

### 7.2 核心类型

```typescript
// note.ts
export interface Note {
  id: string;           // 唯一标识
  slug: string;         // URL slug
  title: string;        // 标题
  description?: string; // 描述
  date?: Date;          // 日期
  category?: string;    // 分类
  tags: string[];       // 标签
  body?: string;        // 内容
}

export interface NoteFrontmatter {
  title?: string;
  description?: string;
  date?: Date;
  category?: string;
  tags?: string[];
}
```

### 7.3 类型使用规范

**导入方式：**
```typescript
// 推荐：通过统一入口
import type { Note, NoteCardData } from '@types';

// 避免：直接导入具体文件
import type { Note } from '../types/note'; // ❌
```

**类型断言：**
```typescript
// 优先使用 satisfies
const config = { } satisfies Config;

// 必要时使用 as
const data = response as Note;
```

## 9. 最佳实践

### 9.1 性能优化

- **图片**：使用 WebP/AVIF 格式，懒加载
- **组件**：使用 `client:visible` 延迟加载非关键组件
- **样式**：利用 Tailwind 的 purge 功能
- **搜索**：使用 Pagefind 预构建索引

### 9.2 代码规范

```typescript
// ✅ 使用可选链和空值合并
const title = note.data?.title ?? 'Untitled';

// ✅ 提前返回减少嵌套
function process(data) {
  if (!data) return null;
  // 处理逻辑
}

// ✅ 使用 Map 进行频繁查找
const cache = new Map();
const value = cache.get(key);

// ❌ 避免深层嵌套
function bad(data) {
  if (data) {
    if (data.items) {
      // ...
    }
  }
}
```

### 9.3 提交规范

```text
feat: 新功能
fix: 修复
docs: 文档
style: 格式调整
refactor: 重构
perf: 性能优化
test: 测试
chore: 构建/工具
```

### 9.4 开发工作流

1. **创建组件**：先在 `ui/` 或 `features/` 创建
2. **添加样式**：在 `components.css` 定义组件类
3. **类型定义**：更新 `types/` 中的相关类型
4. **编写文档**：在组件文件顶部添加 JSDoc
